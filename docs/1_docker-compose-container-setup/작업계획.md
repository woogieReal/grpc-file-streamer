# Docker Compose 컨테이너 환경 구성 작업계획

## 목표

gRPC 파일 스트리머의 실제 전송 로직 구현 전에, Python 기반 서버/클라이언트 컨테이너가 Docker Compose 위에서 정상 기동하고 서로 통신할 수 있는 기반 환경을 구축한다.
proto 파일 정의 및 컴파일 파이프라인까지 포함하여, 이후 기능 구현이 컨테이너 환경 내에서 바로 진행될 수 있도록 한다.

## 작업 범위

- 프로젝트 디렉토리 구조 생성
- proto 파일 작성 (서비스 + 메시지 정의)
- 서버/클라이언트 각각의 Dockerfile, requirements.txt 작성
- docker-compose.yml 작성 (네트워크, 볼륨 마운트 포함)
- 연결 확인용 스켈레톤 main.py 작성
- docker compose up 으로 기동 및 연결 검증

## 세부 작업 목록

### 1단계: 디렉토리 구조 생성

- [x] `proto/` 폴더 생성
- [x] `server/` 폴더 생성
- [x] `client/` 폴더 생성
- [x] `shared/` 폴더 생성 (호스트 마운트 포인트)
  - `shared/input/` — 클라이언트가 읽을 원본 파일
  - `shared/output/` — 서버가 수신 후 저장할 복구 파일

---

### 2단계: proto 파일 작성

- [x] `proto/file_streamer.proto` 작성
  - `FileStreamer` 서비스 정의 (Bidirectional Streaming RPC)
  - `UploadRequest` — `oneof { FileMetadata metadata, bytes chunk }`
    - `oneof` : 같은 스트림에서 메타데이터 패킷과 청크 패킷을 구분하기 위해
  - `UploadResponse` — `oneof { int32 progress, VerificationResult result }`
    - `oneof` : 같은 스트림에서 진행률 피드백과 최종 검증 결과를 구분하기 위해
  - `FileMetadata`, `VerificationResult` 메시지 정의

---

### 3단계: 의존성 정의

- [x] `server/requirements.txt` 작성 — `grpcio`, `grpcio-tools` 버전 고정
  - grpcio-tools는 proto 파일을 컴파일해서 Python stub 코드를 생성하고, grpcio는 그 stub 코드를 런타임에 실행합니다.
  - 두 패키지의 내부 인터페이스가 버전마다 달라지기 때문에, 버전이 다르면 생성된 코드와 런타임이 맞지 않아 임포트 오류 또는 런타임 오류가 발생할 수 있음.
- [x] `client/requirements.txt` 작성 — 동일

---

### 4단계: Dockerfile 작성

- [x] `server/Dockerfile` 작성
  - 베이스: `python:3.12-slim`
  - requirements 설치
  - proto 컴파일 (`grpc_tools.protoc`) 후 Python stub 생성
  - `main.py` 실행
- [x] `client/Dockerfile` 작성
  - 동일 구성

---

### 5단계: docker-compose.yml 작성

- [x] `server` 서비스 정의 — 포트 `50051` 노출, `shared/output` 마운트 (쓰기)
- [x] `client` 서비스 정의 — server hostname으로 연결, `shared/input` 마운트 (읽기)
- [x] 공통 브리지 네트워크 정의
- [x] client가 server 기동 후 연결 시도하도록 `depends_on` 설정

---

### 6단계: 스켈레톤 main.py 작성

- [x] `server/main.py` — `50051` 리슨 후 대기, 기동 로그 출력
- [x] `client/main.py` — 서버에 연결 시도 후 성공/실패 로그 출력

---

### 7단계: 기동 확인

- [x] `docker compose up --build` 실행
- [x] 서버 컨테이너 정상 기동 확인

## 완료 기준

- `docker compose up --build` 실행 시 빌드 오류 없이 두 컨테이너가 기동된다
- 클라이언트 컨테이너 로그에 서버 연결 성공 메시지가 출력된다
- proto 컴파일이 Dockerfile 빌드 단계에서 자동으로 수행된다
