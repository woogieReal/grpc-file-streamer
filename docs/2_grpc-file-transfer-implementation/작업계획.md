# gRPC 파일 전송 로직 구현 작업계획

## 목표

스켈레톤 상태의 서버/클라이언트에 실제 파일 전송 로직을 구현한다.
클라이언트는 `shared/input/`의 파일을 1MB 청크로 분할해 스트리밍 전송하고, 서버는 청크를 수신할 때마다 디스크에 기록하며 진행률을 실시간 반환한다.
전송 완료 후 SHA-256 해시 검증을 통해 파일 무결성을 확인하고 결과를 클라이언트에 반환한다.

## 작업 범위

- `server/main.py` — Upload RPC 핸들러 전체 구현
- `client/main.py` — 파일 읽기 및 스트리밍 전송 전체 구현

---

## 세부 작업 목록

### 1단계: 서버 Upload 핸들러 구현

- [x] 첫 번째 메시지에서 `metadata` 수신 (파일명, 총 크기, 원본 해시)
- [x] 이후 메시지에서 `chunk` 수신 시 `shared/output/{file_name}`에 순차 기록
- [x] 청크 기록 시마다 누적 수신 바이트로 퍼센트 계산 후 `progress` 반환
- [x] EOS(End of Stream) 수신 시 수신 파일 전체 SHA-256 해시 계산
- [x] 원본 해시와 비교 후 `VerificationResult` 반환

---

### 2단계: 클라이언트 전송 로직 구현

- [x] `shared/input/` 디렉토리에서 전송할 파일 탐색
- [x] 파일의 SHA-256 해시 사전 계산
- [x] 첫 패킷으로 `metadata` (파일명, 총 크기, 해시) 전송
- [x] 파일을 1MB 단위로 읽어 `chunk` 패킷으로 연속 전송
- [x] 서버로부터 수신되는 `progress` 실시간 출력
- [x] 최종 `VerificationResult` 수신 후 성공/실패 출력

---

### 3단계: 동작 확인

- [x] `shared/input/`에 테스트 파일 배치
- [x] `docker compose up --build` 실행
- [x] 클라이언트 로그에서 진행률(0~100%) 출력 확인
- [x] 서버 로그에서 파일 저장 및 해시 검증 성공 확인
- [x] `shared/output/`에 수신 파일이 정상 생성됐는지 확인

## 완료 기준

- 클라이언트가 `shared/input/`의 파일을 서버로 전송하고, 진행률이 실시간으로 출력된다
- 전송 완료 후 SHA-256 검증이 성공하고 `VerificationResult(success=True)`가 출력된다
- `shared/output/`에 원본과 동일한 파일이 생성된다
