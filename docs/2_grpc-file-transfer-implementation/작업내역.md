## [2026-05-16] gRPC 파일 전송 로직 구현

### 작업 내용
- `server/main.py` — Upload RPC 핸들러 전체 구현 (metadata 수신 → chunk 기록 → SHA-256 검증 → VerificationResult 반환)
- `client/main.py` — 파일 스트리밍 전송 전체 구현 (SHA-256 사전 계산 → metadata 전송 → 1MB 청크 스트리밍 → progress/result 수신 출력)
- 서버/클라이언트 Dockerfile에 `PYTHONUNBUFFERED=1` 추가

### 주요 결정 사항
- **청크 수신 시 즉시 디스크 기록**: 서버가 모든 청크를 메모리에 모은 뒤 저장하는 대신, 수신 즉시 파일에 기록하여 OOM 없이 1GB 이상 파일도 처리 가능하도록 설계
- **SHA-256을 스트리밍 방식으로 계산**: 클라이언트(사전 계산)와 서버(수신 중 누적) 모두 `hashlib.sha256()`에 청크를 순차적으로 `update()`하여 전체 파일을 메모리에 올리지 않음
- **`iter(lambda: f.read(CHUNK_SIZE), b"")`**: 파일 끝까지 1MB씩 읽는 Python 관용 패턴으로 명시적 루프 없이 청크 제너레이터 구현
- **`WhichOneof()`로 패킷 종류 판별**: proto3 `oneof` 필드의 현재 세팅 여부를 `WhichOneof()`로 확인해 metadata/chunk, progress/result를 명확히 분기

### 이슈 & 해결
- **Docker 환경에서 로그가 출력되지 않음**: Python의 stdout 버퍼링으로 인해 `print()`가 즉시 출력되지 않는 문제 발생. 서버/클라이언트 Dockerfile에 `ENV PYTHONUNBUFFERED=1`을 추가하여 해결
